# Design system

## Status

Visual branding—colour palette, typography, logo, motion style, and component library—is not yet decided. This document defines product constraints that the future design system must honour rather than inventing a brand prematurely.

## Foundations

- Use a consistent spacing, type, colour, radius, and elevation token system.
- Meet WCAG 2.2 AA contrast and keyboard-accessibility expectations where applicable.
- Do not communicate side, outcome, warning level, or time-critical state by colour alone.
- Support readable text at browser zoom and narrow viewports.
- Use motion only to clarify transitions; respect reduced-motion preferences.

## Debate-specific semantics

Public transcript messages, system state, private Lawyer material, and Judge findings must have distinct visual treatments and unambiguous text labels. “Private to you” is always visible in the Lawyer area. A verdict must display its confidence and reasoning context rather than relying on a celebratory or punitive visual alone.

## Content style

Use respectful, direct language. Avoid framing a Judge result as absolute truth; use terms such as “Judge assessment” and show the criteria used. Explain disabled controls—for example, “It is currently the Negative side’s turn”—instead of presenting an unexplained inactive interface.

## Future design decisions

Document approved tokens, fonts, icons, illustration policy, theme support, and reusable primitives here once a visual direction is chosen.
