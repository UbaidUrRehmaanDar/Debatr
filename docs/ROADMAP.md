# Roadmap

## Phase 0 — Documentation and decisions

- Complete the core documentation and resolve the questions required for implementation.
- Align JSON schemas and runtime prompts with the approved specification.
- Define database entities, API contracts, and the debate state machine.

## Phase 1 — Private debate foundation

- Authentication and the small authorised-user model.
- Debate creation, invitations, participant assignment, and state enforcement.
- Turn-based text messaging, timers, and realtime presence.
- Transcript persistence and JSON export/import.

## Phase 2 — AI assistance and judging

- Provider adapter and model configuration.
- Private Lawyer requests with structured, validated responses.
- Judge evaluation, reasoning-first scoring, conduct handling, and reports.
- Pinned facts/evidence context strategy and basic cost controls.

## Phase 3 — Quality and refinement

- End-to-end tests for permissions, state transitions, exports, and AI validation.
- Improve reports, accessibility, observability, and documentation.
- Add only the features needed by the initial private users.

## Deferred

Public debates, matchmaking, leaderboards, ELO, uploads, web research, Fact Checker, Moderator, Coach, voice, teams, tournaments, and mobile are deferred until explicitly prioritised.
